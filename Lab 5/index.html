Preguntas

  1. ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

    Para mejorar la satisfacción del usuario, también ayuda a reducir la carga de procesamiento en el servidor y a notificar de manera instantanea si el usuario cometió algun error, sin esperar respuesta del servidor.

  2. ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

    Existen varias formas, algunas de ellas son: desabilitando el JS dentro del navegador o una es usando el atributo formnovalidate en el botón de submit o poner novalidate

  3. Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

    Porque se validan los datos antes de que lleguen al servidor, así se evita hacer mcuhas peticiones

Preguntas

  • ¿Por qué es importante hacer un session_unset() y luego un session_destroy()?

    session_unset solo borra las variables de sesion actuales, pero no elimina la sesion, por lo que session_destroy remueve la sesion actual

  • ¿Cuál es la diferencia entre una variable de sesión y una cookie?

    En sesión, se mantiene por un periodo corto de tiempo en el servidor, o bien, hasta que se cierre el navegador; mientras que las cookies, siempre que sean aceptadas por el navegador, se mantienen en el cliente (navegador del usuario), hasta que no sea eliminada manualmente por el mismo

  • ¿Qué técnicas se utilizan en sitios como facebook para que el usuario no sobreescriba sus fotos en el sistema de archivos cuando sube una foto con el mismo nombre?

    Si no hay un archivo con el mismo nombre, lo guarda, si sí, le agrega un +1 para que no se guarden con el mismo nombre

  • ¿Qué es CSRF y cómo puede prevenirse?

    Son ataques de ciberdelincuentes CSRF (del inglés Cross Site Request Forgery), tan difíciles de prevenir y detectar que hay que utilizar software avanzados para evitar mayores problemas.

    Es una técnica maliciosa que obliga al navegador web de una víctima, autenticada previamente, a enviar una petición HTTP falsa a una aplicación web vulnerable.

  2. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    GET Solicita un recurso específico ( recupera datos)
    HEAD Pide una respuesta igual a GET,pero sin el cuerpo de la respuesta.
    POST Envia una entidad a un recurso en específico
    PUT Reemplaza todas las representaciones actuales (update) del recurso de destino con la carga útil de la petición.
    PATCH Aplicar modificaciones parciales a un recurso
    DELETE Borra un recurso en específico

Conceptos:

  • URL

    • (Uniform Resource Locator) Conjunto de caracteres identificativos quehacen una dirección única la cual poseen todos los documentos en la World Wide Web (WWW)

  • MIME

    • (Multipurpose Internet Mail Extensions) Manera standard de mandar contenido a través de la red.

  • Métodos y códigos de errores HTTP

    • Los códigos de estado HTTP son respuestas de tres dígitos que un servidor devuelve a petición de un cliente.

    1xx – una respuesta informativa

    2xx – una respuesta exitosa

    3xx – una respuesta de redirección

    4xx – una respuesta de error del cliente

    5xx – una respuesta de error del servidor

  • HTML5

    • Es la última versión de HTML con nuevos elementos, atributos y comportamientos.

  • JS

    • Lenguaje de programación o de secuencias de comandos que te permite implementar funciones complejas en páginas web.

  • php

    • Lenguaje de código abierto muy popular especialmente adecuado para el desarrollo web y que puede ser incrustado en HTML.

  • jsp

    • (Java Server Pages) Tecnología orientada a crear páginas web con programación en Java.

  • Ajax

    • (JavaScript and XML) Conjunto de técnicas que permiten que las aplicaciones web funcionen procesando cualquier solicitud al servidor en segundo plano.
